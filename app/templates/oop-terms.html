GitHub Docs: Repositories

Time to learn about Git, GitHub, & Repositories. Git is a version control software, GitHub is the company that developers make use of cloud-based Git repositories, and Repositories are the storage locations for your previous software archives in your version control process (where your projects are stored)!

Alright, so what is version control? Just as it sounds, it's a safe way to save changes in your code over periods of time without having to overwrite previous versions; in case you have to go back to it later on for some reason. Git counts as distributed version control, meaning each individual developer working with a repository has a copy of it; this is better as centralized as that one entails the "main" project being somewhere specific. Overall, you can expect to initialize or clone a repository, add your changes, commit a snapshot of your code, and push (if you're working alone on a small project, that is)!
