PyTests

PyTest Docs: Get Started

If you've been programming for a while, you've heard of Unit Testing. Unit Testing is arguably more important than the program itself; you do this in order to ensure your program/application is behaving as it should. This, of course, is important in both Python & Web Applications. PyTest is a framework that makes Unit Testing easy. As the name states, it's main use if testing the functionality and behavior of code.

Getting Started

  1. Run the following in your project's command line: pip install -U pytest
  2. Make sure it's installed by running... pytest --version
  3. Create a test_{customName}.py file and add some functions with assertions in it.
  4. Run pytest in the project's command line and you'll see whether they pass or fail.

Testing Simple Pages

Flask Documentation: Testing Applications

Of course, testing Flask Web Applications defers from testing Python command-line applications. There's an entirely different process, but some steps are definitely shared. Here's how I tested the pages for this project:

How-To

  1. Ensure you have a folder named "tests" that is in the root folder of your project.
  2. Create a file named {pageName}_test.py within the tests folder.
  3. If you're going to be checking for pylint later, make sure to include a Docstring at the top.
  4. Create a function with any name that takes "client" its parameter.
  5. Within the function, set response = client.get({page_link}).
  6. Check whether specific text appears on the page via assert b"Specific Words" in response.data
  7. Finally, Check whether the site actually loads with assert response.status_code == 200
  8. Repeat as many times as there are pages. You can also have one for a page that doesn't exist, but assert response.status_code == 404
