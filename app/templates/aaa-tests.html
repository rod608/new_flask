AAA Testing in Python

Arrange, Act, Assert

Jamescooke: Blog

AAA is an acronym that stands for Arrange, Act, and Assert, which in itself is a pattern followed by developers for testing. If you've been programming for a while, you've heard of Unit Testing. In the AAA approach, one focuses each test on a single action. In that sense, it is similar to the Single-Responsibility Principle as you're covering all the bases and making sure each individual component does its one thing well. Let's see how AAA testing is implemented in the following:

  • Arrange: A code block that sets up the conditions for the test action. No assertions should be made. An example of this is the tuple_list in the image above. There's no assertions or actions taking place; just basic setup for what'll be used for testing.
  • Act: Where an action takes place. The instantiation of Multiplication as an object named calculation is the action here, and it has been instantiated using the code block in the arrange section.
  • Assert: Where the actual "testing" takes place. Assertions are made on boolean expressions predicting how the code should behave and throws an error if they don't match. As seen above, the test accurately tests if calculation is an instance of the Multiplication test by making use of the initial Arrangement, Action based on said Arrangement, and final assertion on the behavior.
