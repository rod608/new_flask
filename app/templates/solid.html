SOLID Principles: The Calculator Program

Single Responsibility

Digital Ocean: Definition

Just like Inheritance, the Single-Responsibility Principle is one that is fairly self-explanatory. To sum it up nicely, the principle states that each class should do one thing; and do it well. This way, changes in one place can impact everything in a good way. Troubleshooting will also become a breeze as there's only one place where a specific thing could go wrong. Just taking a look at how the Calculator project's file system is set up shows how it's following this principle. Every major thing is its own class and it does its function well.

Open-Closed

Digital Ocean: Definition

The Open-Closed Principle is also fairly simple; though it is not as transparent as the last. This principle states that completed classes should be open for extension but not for modification, as you don't want to drastically change how it behaves. This means that adding some sort of new feature is allowed as long as it follows the principles and isn't changing any functional behaviors.

Liskov Substitution

Digital Ocean: Definition

Object-Oriented Programming represents real-world relations better than any spaghetti code ever could. Classes like the aforementioned Calculator class have properties and methods; just like the ones in real life. Well, the principle that is Inheritance plays a big role in solidifying the real-world aspects of object-oriented programming by introducing tied relationships. As the name suggests, Inheritance allows for one class to inherit from another. The "child" class inherits all the properties and methods from the parent class and adds its own, meaning the child is dependent on the parent. For example, it would make sense to have a Yorkie class extend a Dog class, seeing as though Yorkies share many similarities with other dogs but throw their own characteristics and classiness into the mix. This is due to the fact that the compiler knows exactly where to call it from, meaning this ties in nicely with the concept that is Encapsulation.
